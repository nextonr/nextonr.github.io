hexo博客搭建指南

Hexo是基于NodeJs的静态博客框架，简单、轻量，其生成的静态网页可以托管在Github和Heroku上。

安装hexo和新建博文

首先自然是安装hexo。

sudo apt-get install nodejs
sudo apt-get install npm
sudo npm install hexo-cli -g
hexo init blog
cd blog
npm install
hexo server
git init

新建博文，markdown格式：

hexo new 第一条博文

或直接在<root>/source/_post目录下新建一个.md文件亦可。

更新完成后重新生成：

hexo clean; hexo generate; hexo server

或：

hexo clean; hexo g; hexo s

配置

修改<root>/_config.yml文件，language写中文——zh，timezone写上海——Asia/Shanghai。坑：进行配置时，需要在冒号:后加一个英文空格

Settings Description
title The title of your website
subtitle The subtitle of your website
description The description of your website
keywords The keywords of your website. Separate multiple keywords with commas ,.
author Your name
language The language of your website. Use a 2-lettter ISO-639-1 code. Default is en.
timezone The timezone of your website. Hexo uses the setting on your computer by default. You can find the list of available timezones here. Some examples are America/New_York, Japan, and UTC.

换主题

hexo可以更换主题，可以在官网查看。我用的是hexo-theme-cactus。

这里涉及一个问题，后面hexo部署到Github上的时候，会自动将生成的静态网页推送到用户的一个特殊仓库，用户名称确定了，该仓库的名称也就确定了。比如用户名为nextonr，该仓库的名称就是nextonr.github.io，部署成功之后，就可以通过域名nextonr.github.io来访问了。但是这个仓库包含的是静态网页，不包含hexo部分的源码以及用户自己写的markdown文件。所以为了解决这个问题，我们要建立两个github仓库：一个公有，托管静态网页；一个私有，托管代码。

但是，theme本身也是一个git仓库，而且需要包含在hexo根目录下，所以我们要使用嵌套的git仓库。注意在使用git submodule命令之前要用git初始化仓库。

git submodule add https://github.com/probberechts/hexo-theme-cactus.git themes/cactus
rm -rf themes/landscape

修改<root>/_config.yml文件，注意冒号后空格，行首不能有空格或制表符：

#theme: landscape
theme: cactus
theme_config:
    colorscheme: white

配置完成之后，重新生成，提交第一次修改：

hexo clean; hexo g
git add .
git commit -m "first commit"

后面还要对hexo进行部署到github上的配置，所以之后再关联远程仓库。

关联远程仓库：

git remote add origin git@github.com:nextonr/blog.git
git push -u origin master

部署到Github

若用户名为nextonr，则需要创建一个名为nextonr.github.io的public仓库。

安装hexo-deployer-git：

npm install hexo-deployer-git --save

配置<root>/_config.yml：

deploy:
    type: git
    repo: git@github.com:nextonr/nextonr.github.io.git
    branch: master

部署：

hexo d

然后我们就可以输入域名nextonr.github.io访问我们的博客了。

hexo根目录关联远程仓库

最后将hexo根目录关联远程仓库，并推送到github上

git remote add origin git@github.com:nextonr/blog.git
git push -u origin master
